## 一、项目背景

### 1.1 产品定位

**Talent Atlas** 是 Trip.com 内部的**智能人才搜索引擎**。

帮助员工（HR、业务主管、项目负责人等）通过自然语言快速找到组织内的人才。

### 1.2 核心痛点

- Trip.com 拥有 **3 万+员工**，组织信息分散在多个系统（Workday、HR Plus、Hive 宽表）
- 人才数据以**长文本为主**（考核评价、晋升记录、peer 反馈、项目经历），司龄越长积累越多，查看困难
- 现有 Talent Card（员工信息卡）堆砌长文本，**信息密度高但可读性差**
- 团队组建、晋升评估、人才盘点等场景都依赖人工搜索，效率低

### 1.3 用户场景

| 场景 | 描述 | 示例 Query |
|------|------|-----------|
| **项目组队** | 找到有特定技能或经验的人 | "做过国际化项目的 7 级以上技术" |
| **晋升评估** | 了解候选人的历史表现和项目背书 | "最近两年晋升过，表现优秀的 6 级" |
| **人才盘点** | 按 BU、序列、地区等维度筛选人才 | "商旅事业部，5 级以上，硕士学历" |
| **内部转岗** | 找到符合目标岗位要求的人 | "搜索推荐部，NLP 方向，6 级" |

### 1.4 现有系统

- **数据层**：员工信息存储在 Hive（edw_emp_ful 宽表），包含结构化字段和长文本字段
- **前端**：已有 Talent Card 组件，但长文本展示不友好
- **技术栈**：React（前端），数据来自 Hive → ES/向量数据库（规划中）

## 二、数据字段说明

### 2.1 结构化字段

| 字段 | 说明 | 示例值 |
|------|------|--------|
| 工号 | 员工唯一标识 | T002847 |
| 姓名 | 中文姓名 | 张明辉 |
| 级别 | 1~8 级，AVP+ | 7 |
| 职务序列 | T=技术/研发，PT=产品，S=客服，P=职能 | T |
| 事业部（BU） | BG-机票事业群、FU-技术中心 等 | BG-机票事业群 |
| 地点 | 上海、北京、深圳、杭州 等 | 上海 |
| 学历 | 博士/硕士/本科/大专/高中 | 硕士 |
| 司龄 | 入职年限 | 7 年 |
| 来源 | 校招 / 社招 | 社招 |
| 是否高学历 | 1=是，0=否 | 1 |
| 是否海外学历 | 1=是，0=否 | 0 |
| 是否海外经验 | 1=是，0=否 | 0 |

### 2.2 长文本字段

| 字段 | 说明 | 特点 |
|------|------|------|
| 项目经历 | 历年参与/主导的项目 | 可能很长（司龄大），格式：年份 + 内容 |
| 考核评价 | 年度绩效评价 | 包含排名、推荐意见、晋升建议 |
| Peer 反馈 | 同事评价 | 自由文本 |
| 晋升记录 | 历史晋升路径 | 格式：年份 级别→级别 |

## 三、核心功能

### 3.1 自然语言搜索

用户在搜索框输入自然语言 Query，系统实时解析并执行混合检索。

**解析示例：**

| 用户输入 | 解析结果 |
|----------|----------|
| 机票事业群，7 以上，做过国际化项目 | 级别 ≥7 + BU=机票事业群 + 语义=国际化项目 |
| 硕士学历以上，有海外经历，6 及以上 | 学历=硕士 + 海外经验 + 级别≥6 |

**搜索框交互：**
- 实时解析：输入后 300ms 自动触发 Query 解析（无需等待回车）
- 解析标签可视化：蓝色=结构化条件，绿色=语义搜索词
- 点击标签可删除（模拟人工修正）

### 3.2 搜索结果列表

- 按匹配度降序排列，显示总分
- 每条结果展示：**姓名 + 级别 + BU + 地点 + 司龄 + 序列**
- 匹配标签高亮：告诉用户"为什么搜到你"
- 圆形分值环：分数 0-100，颜色渐变（绿=高分，橙=低分）

### 3.3 员工详情卡（Talent Card）

点击结果展开详情：

- **Tab 分段**：项目经历 / 考核评价 / Peer 反馈 / 晋升记录
- **长文本折叠**：默认显示 3 行，点击"展开更多"
- **元数据**：工号、来源、入职日期、海外学历/经验标识
- **技能标签**：员工技能标签墙

### 3.4 权限模型

- **搜索**：全员可搜
- **展示**：按行（人员可见性）和列（字段隐私）权限过滤
- Demo 阶段暂不实现权限逻辑，但需预留字段
